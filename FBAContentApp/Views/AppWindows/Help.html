Program Overview



The purpose of this program is to create box content labels that meet Amazon fulfillment centers' specifications (you'll need to log into Seller Central to view the documentation).

The program accepts an Excel workbook (.xlsx extenstion) file as the contents of an entire shipment. Each worksheet in the workbook represents a box. The program goes through each worksheet and creates the appropriate box content string. The program creates a Zebra Programming Language (ZPL) string that is sent to the label printer for printing. (NOTE: The printer used must have the capability of interpreting ZPL 2 commands.) All shipment information is saved to a database and labels like the image below are printed.



The program is loaded with a default Company Ship From address, the one in the image above. In order to reflect your company's information, you must add a company address in the Settings portion of this application. Furthermore, the label also produces the shipment box label that Amazon provides as well. Therefore, you only need this label and not the ones that are generated in Seller Central. However, feel free to use both.

As mentioned previously, when the labels are printed they are also saved to a database. This allows the capability of looking at shipments from the past as well as easily reprinting labels through the History portion of this application. The application also saves shipment contents information in a JSON file to the directory specified in the Settings portion of the application. This is to provide an easy way of doing more with the data if someone chooses to do so.

Creating a Box Content Excel Workbook


This application uses Excel workbooks to read box content information for a shipment. Each worksheet in a workbook represents a box. The box number is determined by the Sheet number. We'll go into more detail on how to prepare this file. If you don't have access to an installation of Excel, you can also prepare this in Google Sheets. As we go through this how-to, examples for both Excel and Google Sheets will be presented.



1. Prepare the file


Using Microsoft Excel

Open a new Excel workbook. Save the workbook. When prompted for the file name, name the file the Shipment Id for the shipment. It usually starts with FBA and a total of 12 characters long. Save it to your desired directory.



Using Google Sheets

From the Google Sheets start page, start a new Blank Sheet.


In the top left-hand corner, title the sheet as the Shipment Id.



2. Fill Sheets with Items


Using your preferred method (using a scanner or manually typing) enter the contents of a box in the first column. Enter each item, even if there's two of the same item. The program will automatically tally-up items with the same identifier.


Example in Excel

Example in Google Sheets


When you're ready for a new box, add a sheet to the workbook and repeat the steps from above.






Repeat this same process for every single box in the shipment.




3. Save and Exit


In Excel this is self-explanatory. File > Save (Ctrl+S) and close the workbook.

In Google sheets there's a few more steps to complete. You'll need to download the workbook as an Excel file. Go to File > Download As > Microsoft Excel (.xlsx). Save to your desired directory.




Now your workbook is ready to be processed by this program. :D




Processing a Shipment



Now that a workbook has been created for a shipment, this program is ready to process it and print box content labels. Before proceeding, make sure you have a default Label Printer selected in Settings, as well as a Company Address, and a Save File Directory. If a Company Address for your company is not set, it will use the default one created at the time of install, which just a generic value. If Label Printer is not set, you will not be able to print thermal labels. However you'll still be able to Print to PDF (Note: You will need to be connected to the internet to render a PDF file.). Lastly, if there is not a Save File Directory set in the Settings of the application, no file will be saved of the contents. However, the shipment will still be saved to the database allowing for reprinting from the History section of the application.

Now that we have that out of the way, let's process a shipment.



1. Add Boxes


From the Main Menu of the application, click on the Process Shipment button. This will take you to the shipment processing page. Click the Add Boxes button and select the workbok that contains the shipment box contents you wish to process. If it's a large shipment with many boxes, give the program a moment to process the file.








2. Select the Amazon Warehouse


Next, select the Amazon Fulfillment Center the shipment is going to. You can find this in Amazon Seller Central. In the Shipping Queue (Seller Central), find your shipment by Shipment ID and look at the column titled "Destination". There will be a four character combination indicating where that shipment is going. That four character combination is in the drop-down menu for selecting an Amazon warehouse. You can also find it by clicking into the shipment from the shipping queue and looking at the "Ship To" address.





If the fulfillment center is not in the drop-down, click the Add New Warehouse button to add it into the database.





3. Print Boxes


Once the boxes have been added and an Amazone Warehouse has been selected, you are ready to print. You have two options for printing the labels; directly to a thermal printer, or generate a PDF file. Regardless of which one you click, the shipment details will save in a JSON formatted file to the Save File Directory as well as saved to the database. A pop-up box will appear informing you of any errors/success.

REMINDER: If there is no defaul thermal printer set in the Settings of the application, nothing will happen if you click Print Thermal Labels. You also have the option of creating a PDF file of the thermal labels. However, you must be connected to the internet because the PDF is returned from Labelry.com's API service.





Amazon Warehouses



Overview


The Amazon Warehouse section lists all the Amazon Warehouses/Fulfillment Centers in the database. The database is pre-loaded with many warehouses located in the U.S. This is not necessarily all of them. From this section you can add, edit, and delete warehouses as needed. Doing so is quite self-explanatory. Any changes you make here are automatically saved to the database.








History



Overview

In this section of the application, a user can view all of the shipments that are stored in the database. Shipments get stored in the database when a shipment is processed and labels are printed from the Process Shipments section of the application. From here you can reprint labels for shipments. You can select one or multiple boxes at a time. You also have the ability to search for a shipment. The search field is case insensitive.

By default, the shipments are ordered by date. The most recent shipment is at the top of the first list box. The second list box contains all the boxes belonging to the selected shipment. To the right of the boxes list box is the shipment details.





You can select as many boxes at a time for reprinting. The print buttons do exactly as labeled. One prints directly to the default label printer configured in Settings. The other renders a PDF file of the box labels to reprint.

Reminder: For PDF's, you must be connected to the internet. Also, if no Label Printer is configured in Settings the labels will not print.

Settings Overview



The Settings section of the application stores all of the default values used throughout the application. The values set here will be used for printing to a thermal printer. Also all files are saved where the Save File Directory points.

Adding a Company Address

Company Address has a loaded default one that's from the database seeding. The address is used to make the labels with the appropriate ship from. When adding a new address, you should fill out the information exactly how your Amazon Seller Central account shows. This way the label that's generated will look like the shipping labels Amazon provides.

To add a new Company Ship From address, simply click the Add New Ship From button and fill out the form in the window that pops up. Again, it's highly recommended to fill out the form with the exact values configured in your Amazon Seller Central account.



Additionally, you can also edit any existing addresses. Simply select it from the drop-down, and click Edit Ship From. The same window will appear with all the details filled out. Make changes as needed. To delete, select the address and click the Delete Ship From button.



Selecting a Label Printer

The drop-down menu for Label Printer is a populated by gathering the name of all the install printers. Select the thermal printer you wish to use. Note: the thermal printer you choose should have the capability of interpreting Zebra Programming Language 2 (ZPL2) commands. If you don't have a thermal printer that supports this, you can always render a PDF file of the labels.




Selecting Save Directory

The Save File Directory is where all files will be saved. The program generates a few different files. When a shipment is processed, a JSON text file of the shipment details is saved. Another text file is also generated which contains the ZPL2 command. It's produced just in case the user would like to see it for whatever reason.

This directory is also where PDF labels are saved when labels are printed to PDF. There's no default place where files will be saved. If this is not set, no files will be set and PDF files will not be able to be generated.
