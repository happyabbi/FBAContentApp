Program Overview



The purpose of this program is to create box content labels that meet Amazon fulfillment centers' specifications (you'll need to log into Seller Central to view the documentation).

The program accepts an Excel workbook (.xlsx extenstion) file as the contents of an entire shipment. Each worksheet in the workbook represents a box. The program goes through each worksheet and creates the appropriate box content string. The program creates a Zebra Programming Language (ZPL) string that is sent to the label printer for printing. (NOTE: The printer used must have the capability of interpreting ZPL 2 commands.) All shipment information is saved to a database and labels like the image below are printed.



The program is loaded with a default Company Ship From address, the one in the image above. In order to reflect your company's information, you must add a company address in the Settings portion of this application. Furthermore, the label also produces the shipment box label that Amazon provides as well. Therefore, you only need this label and not the ones that are generated in Seller Central. However, feel free to use both.

As mentioned previously, when the labels are printed they are also saved to a database. This allows the capability of looking at shipments from the past as well as easily reprinting labels through the History portion of this application. The application also saves shipment contents information in a JSON file to the directory specified in the Settings portion of the application. This is to provide an easy way of doing more with the data if someone chooses to do so.

Creating a Box Content Excel Workbook


This application uses Excel workbooks to read box content information for a shipment. Each worksheet in a workbook represents a box. The box number is determined by the Sheet number. We'll go into more detail on how to prepare this file. If you don't have access to an installation of Excel, you can also prepare this in Google Sheets. As we go through this how-to, examples for both Excel and Google Sheets will be presented.



1. Prepare the file


Using Microsoft Excel

Open a new Excel workbook. Save the workbook. When prompted for the file name, name the file the Shipment Id for the shipment. It usually starts with FBA and a total of 12 characters long. Save it to your desired directory.



Using Google Sheets

From the Google Sheets start page, start a new Blank Sheet.


In the top left-hand corner, title the sheet as the Shipment Id.



2. Fill Sheets with Items


Using your preferred method (using a scanner or manually typing) enter the contents of a box in the first column. Enter each item, even if there's two of the same item. The program will automatically tally-up items with the same identifier.


Example in Excel

Example in Google Sheets


When you're ready for a new box, add a sheet to the workbook and repeat the steps from above.






Repeat this same process for every single box in the shipment.




3. Save and Exit


In Excel this is self-explanatory. File > Save (Ctrl+S) and close the workbook.

In Google sheets there's a few more steps to complete. You'll need to download the workbook as an Excel file. Go to File > Download As > Microsoft Excel (.xlsx). Save to your desired directory.




Now your workbook is ready to be processed by this program. :D




Processing a Shipment



Now that a workbook has been created for a shipment, this program is ready to process it and print box content labels. Before proceeding, make sure you have a default Label Printer selected in Settings, as well as a Company Address, and a Save File Directory. If a Company Address for your company is not set, it will use the default one created at the time of install, which just a generic value. If Label Printer and Save File Directory are not set, the labels will not print and no file will be saved of the shipment details. However, the program will still save it to the database and you will be able to view the shipment in the History section of the application.

Now that we have that out of the way, let's process a shipment.



1. Add Boxes


From the Main Menu of the application, click on the Process Shipment button. This will take you to the shipment processing page. Click the Add Boxes button and select the workbok that contains the shipment box contents you wish to process. If it's a large shipment with many boxes, give the program a moment to process the file.








2. Select the Amazon Warehouse


Next, select the Amazon Fulfillment Center the shipment is going to. You can find this in Amazon Seller Central. In the Shipping Queue (Seller Central), find your shipment by Shipment ID and look at the column titled "Destination". There will be a four character combination indicating where that shipment is going. That four character combination is in the drop-down menu for selecting an Amazon warehouse. You can also find it by clicking into the shipment from the shipping queue and looking at the "Ship To" address.





If the fulfillment center is not in the drop-down, click the Add New Warehouse button to add it into the database.





3. Print Boxes


Once the boxes have been added and an Amazone Warehouse has been selected, you are ready to print. Simply click the Print Boxes button and the labels will print as well as saving the shipment details in a JSON formatted file and to the database. A pop-up box will appear informing you of any errors/success.





REMINDER: If a Company Address for your company is not set, it will use the default one created at the time of install, which just a generic value. If Label Printer and Save File Directory are not set, the labels will not print and no file will be saved of the shipment details. However, the program will still save it to the database and you will be able to view the shipment in the History section of the application.




Amazon Warehouses



Overview


The Amazon Warehouse section lists all the Amazon Warehouses/Fulfillment Centers in the database. The database is pre-loaded with many warehouses located in the U.S. This is not necessarily all of them. From this section you can add, edit, and delete warehouses as needed. Doing so is quite self-explanatory. Any changes you make here are automatically saved to the database.








History



Overview

In this section of the application, a user can view all of the shipments that are stored in the database. Shipments get stored in the database when a shipment is processed and labels are printed from the Process Shipments section of the application.
